File .lck trong Ảo hóa VMware

Hiểu rõ cơ chế khóa file và cách xử lý khi bị kẹt (stale lock)

1. File .lck là gì?

File .lck (lock file) là file khóa được VMware tạo ra để ngăn nhiều process truy cập đồng thời vào cùng một file đĩa ảo (VMDK) hoặc tài nguyên khác, nhằm tránh data corruption.

Chủ yếu xuất hiện trên:
VMware ESXi / vSphere • VMware Workstation • VMware Fusion • VMware Player

2. Khi nào file .lck xuất hiện?

  • Khi máy ảo (VM) đang chạy hoặc đang được mở
  • Khi tạo hoặc xóa snapshot
  • Khi backup bằng công cụ thứ 3 (Veeam, Nakivo, vSphere Data Protection…)
  • Khi mount đĩa VMDK vào một VM khác
  • Khi suspend VM (tạo file .vmss)

Cấu trúc thư mục điển hình

MyVM.vmx
MyVM.vmdk
MyVM-flat.vmdk
MyVM-000001.vmdk
MyVM-000001-delta.vmdk
lck-12345678-abcdef/      ← Thư mục khóa
    M12345678.lck
    Mabcdef.lck

3. Bình thường hay có vấn đề?

Bình thường: Máy ảo đang chạy → luôn có file .lck
→ Khi tắt VM đúng cách, file .lck sẽ tự động bị xóa
Có vấn đề (stale/orphaned lock):
  • VM đã tắt hoặc crash
  • Host ESXi bị crash/reboot đột ngột
  • vSphere Client bị đóng đột ngột
  • Backup tool treo, không release lock
  • Storage mất kết nối tạm thời

Hậu quả: Không power on được VM, báo lỗi:

  • “File is locked by another process”
  • “Cannot open the disk … or one of the snapshot disks it depends on”
  • “The file is in use by another running virtual machine”

4. Cách xử lý file .lck bị kẹt (Stale Lock)

Cách 1: Khởi động lại host (an toàn nhất)

ESXi: Restart management agents hoặc reboot host
Workstation/Fusion: Đóng hoàn toàn VMware → mở lại

Cách 2: Kill process trên Windows (Workstation/Fusion)

  1. Mở Resource Monitor → tab CPU → Associated Handles
  2. Tìm process vmware-vmx.exe hoặc vmware.exe
  3. Tìm handle chứa đường dẫn thư mục lck-...
  4. Right-click → End Process

Cách 3: Xử lý trên ESXi qua SSH (phổ biến nhất)

# Đăng nhập SSH vào host ESXi

# Xem danh sách VM
vim-cmd vmsvc/getallvms

# Tìm process đang giữ file (nếu có)
lsof /vmfs/volumes/.../TenVM/TenVM.vmdk

# Tìm tất cả file .lck
find /vmfs/volumes/ -name "*.lck" -ls

# Nếu không có process nào dùng → xóa thủ công
rm -rf /vmfs/volumes/.../TenVM/lck-*

Cách 4: Storage vMotion hoặc Unregister/Register

  • Storage vMotion → tự động giải phóng lock cũ
  • Unregister VM → Browse Datastore → Right-click .vmx → Register lại

5. Mẹo tránh file .lck bị kẹt

  • Luôn shutdown guest thay vì power off cứng
  • Không đóng đột ngột vSphere Client/Web Client khi đang thao tác
  • Sử dụng backup tool hỗ trợ VMware tốt (Veeam, Nakivo…)
  • Tránh dùng NFS latency cao → dễ mất lock tạm thời
  • Cấu hình timeout hợp lý cho storage

Tóm lại

File .lck là cơ chế bảo vệ bình thường của VMware.
Chỉ trở thành vấn đề khi nó còn tồn tại mà không có process nào đang sử dụng → lúc đó mới cần can thiệp thủ công như trên.
